# Устранение проблем с SSH подключением

## Проблема: Connection timed out

Это означает, что порт 22 (SSH) закрыт или VM недоступна.

## Решения:

### 1. Проверьте IP адрес VM

В консоли Yandex Cloud:
- Перейдите в **Compute Cloud** → **Виртуальные машины**
- Найдите вашу VM
- Проверьте **Публичный IPv4** адрес
- Убедитесь, что адрес правильный: `158.160.180.232`

### 2. Проверьте Security Groups

В консоли Yandex Cloud:
- Перейдите в **VPC** → **Security Groups**
- Найдите Security Group, привязанную к вашей VM
- Убедитесь, что есть правило:
  - **Тип**: SSH
  - **Порт**: 22
  - **Протокол**: TCP
  - **Источник**: 0.0.0.0/0 (или ваш IP)

### 3. Проверьте статус VM

В консоли Yandex Cloud:
- Убедитесь, что VM **запущена** (статус "Running")
- Если остановлена - запустите её

### 4. Подключение через Cloud Shell (обходной путь)

Если SSH не работает, используйте Cloud Shell:

1. **Откройте Cloud Shell** в консоли Yandex Cloud (кнопка в правом верхнем углу)

2. **Подключитесь к VM из Cloud Shell:**
   ```bash
   ssh admin_t_t_r@158.160.180.232
   ```
   
   Или если нужен ключ:
   ```bash
   ssh -i ~/.ssh/yandex_vm_key admin_t_t_r@158.160.180.232
   ```

3. **После подключения** выполняйте команды на VM

### 5. Альтернатива: Используйте веб-терминал Yandex Cloud

В консоли Yandex Cloud:
- Перейдите к вашей VM
- Нажмите кнопку **"Подключиться"** или **"Terminal"**
- Откроется веб-терминал прямо в браузере

## Быстрая проверка:

Выполните в Cloud Shell:
```bash
# Проверка доступности порта 22
nc -zv 158.160.180.232 22

# Или
telnet 158.160.180.232 22
```

Если порт недоступен - проблема в Security Groups или VM остановлена.
