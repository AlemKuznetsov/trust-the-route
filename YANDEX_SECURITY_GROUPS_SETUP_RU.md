# Настройка Security Groups в Yandex Cloud (на русском)

## Как открыть порт 8080 для backend API:

### Способ 1: Через раздел VPC (рекомендуется)

1. **В левом меню консоли Yandex Cloud** найдите раздел **"VPC"** (Virtual Private Cloud)
   - Обычно находится в разделе "Сеть" или "Network"

2. **Перейдите в "Security Groups"** (Группы безопасности)

3. **Найдите Security Group**, привязанную к вашей VM:
   - В списке Security Groups найдите ту, которая используется вашей VM
   - Или откройте вашу VM → раздел "Сеть" → там будет указана Security Group

4. **Нажмите на Security Group** для редактирования

5. **Нажмите "Добавить правило"** или "Add rule"

6. **Настройте правило:**
   - **Направление**: Входящий (Inbound)
   - **Протокол**: TCP
   - **Порты**: 8080
   - **Источник**: CIDR
   - **CIDR блоки**: `0.0.0.0/0` (для доступа из интернета)
     - Или укажите ваш IP для большей безопасности
   - **Описание**: "Backend API порт 8080"

7. **Сохраните правило**

### Способ 2: Через раздел VM (если есть быстрый доступ)

1. **Откройте вашу VM** (compute-vm-2-4-25-ssd-1769660811014)

2. **Перейдите в раздел "Сеть"** или "Network"

3. **Найдите Security Group** и нажмите на неё

4. **Добавьте правило** как описано выше

## Проверка:

После добавления правила, проверьте доступность порта:

```bash
# С вашего компьютера (PowerShell)
Test-NetConnection -ComputerName 158.160.217.181 -Port 8080

# Или через curl
curl -v http://158.160.217.181:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{\"email\":\"test@example.com\",\"password\":\"test123\",\"name\":\"Test\"}'
```

## Важно:

⚠️ **Обратите внимание**: В скриншоте виден IP адрес `158.160.217.181`, а не `158.160.180.232`, который мы использовали ранее. 

**Проверьте правильный IP адрес вашей VM:**
- В консоли Yandex Cloud → ваша VM → раздел "Обзор" или "Доступ"
- Найдите "Публичный IPv4" адрес

**И обновите в Android приложении:**
- Файл: `app/src/main/java/com/trusttheroute/app/di/NetworkModule.kt`
- Измените `BASE_URL` на правильный IP
